    The Vienna Ab initio Simulation Package (VASP) is a computer program for atomic scale materials modelling. VASP users are widely distributed, such as chemistry, physics, material, enviornment. Users have developed and contributed scripts or applets used in their field based on VASP. VASPKIT is a Pre- and Post-Processing Program for the VASP code, written by Fortran and used under the Linux environment.

    1. Quick Start

    (1) In order to ensure the full functionality of the VASPKIT, users need to configure the environment of the vaspkit from ~/.vaspkit file. Copy the environment variable file to the user directory by running the following command under the bash terminal.

    Edit the .vaspkit file,

    The configuration file is mainly used to set the environment variables of the vaspkit, including the VASP version information, the directory of the target library, the selection of the functional, and whether to select the POTCAR according to the VASP official recommendation, and so on.

    (2) Setting up local repository pseudopotential directory. For example, unzip the VASP PAW_PBE pseudopotentials at ‘~/POTCAR/PBE’.

    (3) Add the absolute path of the vaspkit to the environment variable.

    Replace the absolute path of the /home/vaspkit.0.73/bin/ directory by user’s own vaspkit executable.

    Vaspkit provides an automatic configuration script setup.sh since version 0.73. Execute

    ​ source setup.sh

    or bash setup.sh

    If ~/.vaspkit already exists, the above operation will not overwrite it and still use the old version of the environment variable.

    2. Generate Input files

    In order to successfully run VASP tasks, usually one need 4 files: INCAR, POSCAR, POTCAR and KPOINTS.

    1. INCAR file is a control file that tells VASP what tasks to calculate and how to calculate;

    2. POSCAR is a file containing lattice information, atomic coordinate information, and atomic velocity information (for MD);

    3. POTCAR is a pseudopotential file, which is USPP or PAW type;
    4. KPOINTS, which can be included in INCAR, but not recommended for omitting, contains K-points information in the reciprocal space at which the wave function integrates to obtain the charge density.
